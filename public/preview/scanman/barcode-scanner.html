How it works

Scan → Parse → Validate → Route → Act

Every barcode follows the same pipeline, regardless of symbology or warehouse operation.

01

Capture

Camera, hardware scanner, or manual entry. Supports GS1-128, EAN-13, DataMatrix, QR, Code 128, ITF-14, PDF417

02

Detect

Auto-detect symbology and type. Strip FNC1 prefix (]C1, ]d2, ]e0, ]Q3). Identify Application Identifiers

03

Parse

Walk AI codes (2/3/4-digit). Extract GTIN, SSCC, batch, expiry, serial, weight. Handle fixed & variable length AIs

04

Validate

Mod-10 check digit for GTIN-8/12/13/14 and SSCC-18. Date validation for YYMMDD AIs. Structure checks

05

Route

Match to current mode: receiving checks PO, picking validates pick list, shipping confirms manifest. Audio + haptic feedback

Scanning skill paths

Six workflows, one scanner

Each warehouse operation has specific barcode expectations and validation rules. The scanner adapts its parsing, validation, and feedback to the active workflow.

📦

Inbound Receiving

Scan GS1-128 or ITF-14 on incoming cartons and pallets. Parse SSCC from ASN labels. Extract GTIN, batch/lot, expiry, and quantity. Match against Purchase Order.

GS1-128 ITF-14 EAN-13 DataMatrix
🏗️

Put-Away & Storage

Scan item barcode for product confirmation, then scan Code 128 location label for bin assignment. Record zone placement in WMS. Enforce FIFO and zone rules.

Code 128 Code 39 EAN-13
🛒

Order Picking

Scan location first to confirm correct aisle/bin. Then scan item GTIN to validate against pick list. Supports wave picking, batch picking, and zone-based picking.

Code 128 EAN-13 UPC-A DataMatrix
📋

Packing & Cartonisation

Scan items into shipping carton. Generate and scan SSCC for outbound carton. Link carton contents to SSCC for traceability. Capture weight via AI (310n).

EAN-13 GS1-128 Code 128
🚛

Shipping & Dispatch

Scan SSCC on each pallet/carton being loaded. Validate against shipment manifest. Record trailer assignment. Read carrier labels (MaxiCode for UPS, PDF417 for BOL).

GS1-128 PDF417 MaxiCode
📊

Cycle Counting

Scan location barcode to begin count. Scan each item at location. Record quantities, flag discrepancies against WMS records. Support blind counts and guided counts.

Code 128 EAN-13 UPC-A

Technology stack

What powers the scanner

html5-qrcode

Primary scanning library (ZXing-js wrapper). All 1D + 2D formats. Camera + file input. CDN-loaded.

Quagga2

Alternative for 1D-only. Better rotation invariance for angled/curved barcode surfaces on corrugated cardboard.

BarcodeDetector API

Native browser API (Chrome/Edge 83+). Zero dependencies for managed Android warehouse devices.

Web Audio API

Synthesised beep/buzz feedback tones. No audio files required. Distinct sounds for success, error, duplicate.

React 18

Component architecture. useRef for scanner lifecycle. useCallback for stable scan handlers. CDN-loaded.

GS1 Digital Link

URL-encoded product data in QR codes. Sunset 2027 transition support for dual 1D + 2D barcodes.
